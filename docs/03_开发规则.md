# 开发规则

1. 模块化开发。
2. 不修改无关模块。
3. 大功能拆分小任务。
4. 修改前先分析。
5. 修改后更新日志。

任何 AI 接手项目时，先阅读项目文档。

---

## 附注 · 落地细则（小克按现状整理）

- 新功能 = `src/modules/` 新增一个目录（xxxService.js + XxxPage.jsx），在 `core/router.jsx` 注册路由
- 新主题 = `src/themes/` 新增一个 CSS 文件，新样式必须用 `var(--xxx)` 变量而非写死颜色
- 不删除已有页面和功能；不创建新的替代项目

### Git 约定

- 在功能分支上开发，不直接推 main
- 提交信息用中文，说清楚做了什么
- 合并到 main 后，GitHub Actions 自动构建部署到 GitHub Pages

### 验证要求

- 改完必须 `npm run build` 确认构建通过
- UI 改动尽量截图验证（桌面 + 移动两种宽度、四套主题）
