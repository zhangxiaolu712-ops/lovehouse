# 03 · 开发规则

任何 AI（或人）修改本项目，必须遵守以下规则：

1. **先读 `CLAUDE.md`**，了解项目当前状态、身份信息和记忆系统
2. **不创建新的替代项目**，在现有结构上修改
3. **不删除已有页面和功能**
4. **新功能以模块形式添加**到 `src/modules/`（一个目录 = Service + Page 两个文件）
5. **新主题以 CSS 文件形式添加**到 `src/themes/`
6. **修改前先说明方案**，得到确认或依据明确指示再动手
7. **修改后更新 `CLAUDE.md` 的变更记录**（格式见 `04_搭建日志模板.md`）

## Git 约定

- 在功能分支上开发，不直接推 main
- 提交信息用中文，说清楚做了什么
- 合并到 main 后，GitHub Actions 自动构建部署到 GitHub Pages

## 验证要求

- 改完必须 `npm run build` 确认构建通过
- UI 改动尽量截图验证（桌面 + 移动两种宽度）
- 不同主题下都要正常（四套主题共用 CSS 变量，新样式必须用 `var(--xxx)` 而非写死颜色）
