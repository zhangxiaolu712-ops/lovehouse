# LoveHouse 当前架构

一级模块：

- 空间中心
- 记忆中心
- AI中心
- 设备中心
- 项目中心
- 设置中心

设计理念：

**首页是房间。大模块是柜子。小功能是抽屉。**

---

## 附注 · 六大中心与抽屉对应表（已落地为侧边栏导航）

占位抽屉统一由 `modules/placeholder/PlaceholderPage.jsx` 渲染（显示「规划中」+ 功能介绍），做好一个换一个。

## 附注 · 代码目录结构

```
lovehouse/
├── src/
│   ├── main.jsx                 ← 应用入口
│   ├── core/                    ← 核心零件（可单独替换）
│   │   ├── supabase.js          ← Supabase 客户端（换数据库只改这里）
│   │   ├── theme.jsx            ← 主题切换（Context + localStorage）
│   │   └── router.jsx           ← 路由注册表
│   ├── modules/                 ← 功能模块 = 抽屉（每个独立，互不依赖）
│   │   ├── diary/ memory/ quotes/ todo/ mood/ stream/ changelog/
│   │   └── （每个模块 = xxxService.js 数据层 + XxxPage.jsx 页面）
│   ├── themes/                  ← 主题样式（classic/cozy/vintage/desktop）
│   └── shared/
│       ├── AppShell.jsx         ← 应用外壳（桌面侧边栏 + 移动底部导航）
│       ├── Home.jsx             ← 首页（房间）
│       └── global.css           ← 全局样式
├── .env / .env.production       ← Supabase 配置
└── CLAUDE.md / docs/ / legacy/
```

## 附注 · 布局（响应式侧边栏格局）

- 桌面端（≥768px）：左侧固定侧边栏 — 品牌、恋爱天数、8 个页面导航、主题快捷切换
- 移动端（<768px）：底部导航栏，侧边栏隐藏

## 附注 · 数据库表

| 表名 | 字段 | 说明 |
|------|------|------|
| memories | id, content, category, importance, level, created_at | 记忆碎片，level 分固定/长期/短期/临时 |
| diary | id, title, content, mood, created_at | 日记本 |
| quotes | id, content, speaker, created_at | 语录墙 |
| todo | id, content, done, created_at | 待办事项 |
| mood_log | id, mood, note, created_at | 心情日志 |
| stream | id, title, content, created_at | 动态流 |
| notes | id, content, author, color, created_at | 小纸条留言板，author 默认"小婷"，color 默认"pink" |

注意：所有表 RLS 当前关闭（个人项目暂可接受，待开发列表中有加固计划）。
