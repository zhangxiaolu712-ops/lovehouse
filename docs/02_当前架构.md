# 02 · 当前架构

```
lovehouse/
├── .env                         ← Supabase 密钥（不提交，本地开发用）

│
├── src/
│   ├── main.jsx                 ← 应用入口
│   ├── core/                    ← 核心零件（可单独替换）
│   │   ├── supabase.js          ← Supabase 客户端
│   │   ├── theme.jsx            ← 主题切换（Context + localStorage）

│   │   ├── diary/     📖 日记
│   │   ├── memory/    💎 记忆
│   │   ├── quotes/    💬 语录

│   │   ├── mood/      🌈 心情
│   │   ├── stream/    🌊 动态流
│   │   └── changelog/ 📋 搭建日志
│   ├── themes/                  ← 主题样式（每套独立 CSS）
│   │   ├── classic/   🌸 恋爱小屋
│   │   ├── cozy/      💙 浪漫蓝
│   │   ├── vintage/   📜 复古手账
│   │   └── desktop/   🌙 夜空紫

│       ├── Home.jsx             ← 首页
│       └── global.css           ← 全局样式
```

## 模块内部约定

每个功能模块包含两个文件：

- `xxxService.js` — 数据层，封装对 Supabase 的 CRUD，页面不直接碰数据库
- `XxxPage.jsx` — 页面组件，在 `core/router.jsx` 注册路由

## 布局（响应式侧边栏格局）

- **桌面端（≥768px）**：左侧固定侧边栏 — 品牌标志、恋爱天数、8 个页面导航、主题快捷切换圆点；内容区最宽 640px
- **移动端（<768px）**：底部导航栏（6 个常用入口），侧边栏隐藏

## 数据库表

| 表名 | 字段 | 说明 |
|------|------|------|
| memories | id, content, category, importance, level, created_at | 记忆碎片，level 分固定/长期/短期/临时 |
| diary | id, title, content, mood, created_at | 日记本 |
| quotes | id, content, speaker, created_at | 语录墙 |
| todo | id, content, done, created_at | 待办事项 |
| mood_log | id, mood, note, created_at | 心情日志 |
| stream | id, title, content, created_at | 动态流 |

注意：所有表 RLS 当前关闭（个人项目暂可接受，待开发列表中有加固计划）。
